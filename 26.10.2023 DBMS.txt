  <h4 class="head1"> 1. What is DBMS ?</h4>
  <p class="a1">DBMS stands for "Database Management System." It is software that is designed to efficiently and securely manage, store, retrieve, and manipulate data in a structured way. DBMS serves as an intermediary between the user and the database, providing an organized and structured environment for managing data .</p>
<center>
    <iframe width="560" height="315" src="https://www.youtube.com/embed/T7AxM7Vqvaw?si=FPUoWw2qrcx9Yr1U" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen></iframe></center></br>
  <p class="a2">DBMS Provides a tool  for performing various operation such as inserting , deleting,updating into a database.</p>
  
   <h5 class="headA"> Key features of a DBMS include:</h5>
  <h5 class="headB">1.Data Storage:</h5><p class="obj1"> DBMS stores data in an organized and structured manner, typically using tables with rows and columns. This allows for efficient data retrieval and management.

  <h5 class="headB">2.Data Retrieval:</h5><p class="obj1"> Users can query the database to retrieve specific data using query languages like SQL (Structured Query Language). DBMS optimizes the retrieval process for speed and accuracy.

  <h5 class="headB">3.Data Security:</h5><p class="obj1"> DBMS systems provide mechanisms for securing data through access controls, authentication, and authorization. This ensures that only authorized users can access and modify the data.

  <h5 class="headB">4.Data Integrity:</h5><p class="obj1"> DBMS enforces data integrity rules to maintain the consistency and accuracy of data. This includes enforcing constraints, such as unique keys and foreign keys, to prevent data anomalies.

  <h5 class="headB">5.Concurrency Control:</h5><p class="obj1"> In multi-user environments, DBMS manages concurrent access to the database to prevent conflicts and ensure data consistency.

<h5 class="headB">6.Data Backup and Recovery:</h5><p class="obj1"> DBMS systems offer tools for data backup and recovery to protect against data loss due to hardware failures or other unforeseen events.

<h5 class="headB">7.Data Indexing:</h5><p class="obj1"> DBMS systems create indexes to improve the speed of data retrieval by creating data structures that allow for faster searching.

  <h5 class="headB">8.Transaction Management:</h5><p class="obj1"> DBMS systems support transactions, which are sequences of database operations that are treated as a single unit. Transactions ensure data consistency and reliability.
  </br>

<h5 class="headD">There are four types of Data languages</h5>
<P class="headE">
  1.Data Definition Language (DDL) :</P>
<p class="obj3">DDL statements are used to define and manage the structure of database objects such as tables, indexes, and constraints.</br>
DDL is the short name for Data Definition Language, which deals with database schemas and descriptions, of how the data should reside in the database.
</br></br>
<h5 class="headF">(i) CREATE: </h5>
<p class="obj4"> It's Used for creating database objects like tables, indexes, and views.These objects are crucial for structuring and organizing data within the database.</br></br>
<h5 class="headF">(ii) ALTER: </h5>
<p class="obj4">This statement modifies the structure of an existing table, like adding or removing columns.</br></br>
<h5 class="headF">(iii) DROP: </h5>
<p class="obj4"> DROP statement is used to permanently remove database objects such as tables, indexes, views, or even entire databases.</br></br>
<h5 class="headF">(iv)TRUNCATE: </h5>
<p class="obj4"> This statement used to quickly and efficiently remove all rows from a table, effectively deleting all data in the table while keeping the table structure intact. </br>
Unlike the DELETE statement, which removes rows one by one and logs each deletion, TRUNCATE operates as a bulk operation and is generally faster.</br></br>
<h5 class="headF">(v)COMMENT: </h5> 
 <p class="obj4">The COMMENT command is used to add comments or descriptions to various database objects, such as tables, columns, or views. These comments can provide additional information or documentation about the objects.</br></br>
<h5 class="headF">(vi)RENAME:</h5><p class="obj4">The RENAME command is used to change the name of an existing database object, such as a table, column, or index.
Renaming can be useful for various purposes, including reorganizing the database schema or making object names more meaningful.
</br></br>

<p class="headG">2.Data Manipulation Language(DML): </P>
<p class="obj5">DML statements are used to interact with the data stored in the database. They include operations like inserting, updating, and deleting data.</br></br>
(i)SELECT: retrieve data from a database</br></br>
(ii)INSERT: insert data into a table</br></br>
(iii)UPDATE: updates existing data within a table</br></br>
(iv)DELETE: Delete all records from a database table</br></br>
(v)MERGE: UPSERT operation (insert or update)</br></br>
(vi)CALL: call a PL/SQL or Java subprogram</br></br>
(vii)EXPLAIN PLAN: interpretation of the data access path</br></br>
(viii)LOCK TABLE: concurrency Control
<p class="headH"> 3.Data Control Language (DCL) :</p>
<p class="obj6">In a database management system DCL stands for Data Control Language. DCL is a subset of SQL (Structured Query Language) that is used to control access to data within a database. It includes two main types of SQL commands:</br></br>
<b>1. GRANT :</b><p class="para1">The GRANT command is used to give permissions to database users or roles. <br>This is use to read, write, update, or delete the data in a certain database tables,<br> as well as execute specific stored procedures or functions. GRANT allows database administrators to control who can perform various actions on the database.</p>
<p class="obj6">
<b>2.REVOKE :</b><p class="para1">It is used to remove the privileges on user accounts for access to a database object , <br>It allows administrators to revoke specific permissions from a user or role. <br><br> Example :-<br><p class="para2"> REVOKE SELECT ON employee FROM user1;</br><p class="para3">This command will REVOKE a SELECT privilege on employee table from user1.</p>
<p class="headI"> 4.Transaction Control Language(TCL) :</p>
<p class="obj7">TCL is a subset of SQL (Structured Query Language) used to manage transactions in a database. <br>TCL commands are typically used in multi-user environments where multiple transactions may be occurring simultaneously.
  <br><br>There are two primary TCL Commands :</br></br>
<b>COMMIT :</b><p class="para1">The COMMIT command is used to save all the changes made during a transaction to the database.<br> Once a COMMIT is issued, the changes become permanent and cannot be rolled back. </br></br>
<p class="obj7">
<b>ROLLBACK :</b><p class="para1">The ROLLBACK command is used to undo all the changes made during a transaction and restore <br>the database to its state before the transaction started. It is typically used when an error occurs<br> within a transaction or when a transaction needs to be canceled.
  
  



  
  
  
  
  
  
  
  
  
  
  
    <h4 class="head2">2. What is RDBMS ?</h4>
    <p class="a3">RDBMS stands for Relational Database Management System. It is a type of database management system that organizes and stores data in a structured manner based on the principles of relational databases. RDBMS software is designed to manage and manipulate data efficiently, ensuring data integrity,consistency, and reliability. Here are some key characteristics and components of an RDBMS:
    </br></br>
<h5 class="head3">1. Tables:</h5>
<p class="a4">Data in an RDBMS is organized into tables, which are also called relations. Each table consists of rows and columns, with each row representing a record or a tuple, and each column representing a specific attribute or field.
  </br>
<h5 class="head4">2. Data Integrity: </h5> <p class="a5">RDBMS systems enforce data integrity through constraints, such as primary keys, foreign keys, unique constraints, and check constraints. These constraints help maintain data accuracy and prevent inconsistencies.
 </br>
<h5 class="head6">3. Structured Query Language (SQL):</h5><p class="a7"> RDBMS systems use SQL as the standard language for defining, querying, and manipulating data. SQL allows users to perform operations like SELECT (retrieve data), INSERT (add new data), UPDATE (modify existing data), and DELETE (remove data) on the database.
 </br></br>
<h5 class="head6">4. ACID Properties:</h5><p class="a7"> RDBMS systems adhere to the ACID (Atomicity, Consistency, Isolation, Durability) properties to ensure data reliability and transactional consistency. Transactions are treated as atomic units, meaning they are either fully completed or fully rolled back in case of a failure.
 </br></br>
<h5 class="head7">5. Relationships:</h5><p class="a8"> RDBMS systems allow the establishment of relationships between tables using foreign keys. These relationships define how data in different tables is related, enabling complex data modeling and retrieval.
 </br>
<h5 class="head8">6. Normalization:</h5><p class="a9"> RDBMS encourages data normalization, a process use to reduce data redundancy and improve data consistency. Normalization helps in efficient storage and maintenance of data.
 </br></br>
<h5 class="head9">7. Indexing:</h5><p class="a10"> To speed up data retrieval, RDBMS systems often use indexing mechanisms. Indexes provide a way to quickly locate records based on specific columns, improving query performance.
 </br></br>
<h5 class="head10">8. Multi-User Support:</h5><p class="a11"> RDBMS systems are designed to support multiple concurrent users, ensuring data consistency and preventing data conflicts through locking and transaction management.
 </br></br>
<h5 class="head11">9. Security:</h5> <p class="a12">RDBMS systems offer security features to control access to the data, including user authentication, authorization, and encryption of sensitive data.
 </br></br>
<h5 class="head12">10. Scalability:</h5> <p class="a13">RDBMS systems can be scaled vertically (by increasing hardware resources) or horizontally (by adding more database servers) to accommodate growing data and user loads.
 </br></br>
Popular examples of RDBMS software include MySQL, PostgreSQL, Microsoft SQL Server, Oracle Database, and SQLite. These systems are widely used in various applications, from simple web applications to complex enterprise-level systems, where structured and organized data management is crucial.
